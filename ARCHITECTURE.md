# Step Stats — Architecture Overview

## Tech Stack

| Layer      | Technology                  | Rationale                                                    |
| ---------- | --------------------------- | ------------------------------------------------------------ |
| Backend    | **PocketBase**              | Single-binary backend with SQLite, REST API, static file serving |
| Frontend   | **Alpine.js + Tailwind CSS** | Lightweight, no build step, served directly from `pb_public/` |
| Charts     | **Chart.js**                | Simple, responsive charting with good mobile support          |
| Deployment | **Docker (single container)** | PocketBase binary + static frontend, ARM-compatible for Raspberry Pi |

All frontend dependencies loaded via CDN — **zero build tooling required**.

---

## Data Model

### Collection: `steps`

| Field   | Type   | Constraints         | Description                |
| ------- | ------ | ------------------- | -------------------------- |
| `date`  | Date   | Required, Unique    | The calendar date           |
| `count` | Number | Required, Min: 0    | Total steps for that day    |

### Collection: `goals`

| Field    | Type   | Constraints         | Description                          |
| -------- | ------ | ------------------- | ------------------------------------ |
| `year`   | Number | Required, Unique    | Calendar year (e.g. 2026)            |
| `target` | Number | Required, Min: 0    | Total step target for the entire year |

**Derived values (computed in frontend):**
- **Daily goal** = `target / days_in_year` (365 or 366)
- **Monthly goal** = `daily_goal × days_in_month`

## API (auto-generated by PocketBase)

All standard REST endpoints under `/api/collections/{collection}/records`:

| Operation | Steps                                   | Goals                                   |
| --------- | --------------------------------------- | --------------------------------------- |
| List      | `GET /api/collections/steps/records`    | `GET /api/collections/goals/records`    |
| Create    | `POST /api/collections/steps/records`   | `POST /api/collections/goals/records`   |
| Read      | `GET /api/collections/steps/records/:id`| `GET /api/collections/goals/records/:id`|
| Update    | `PATCH /api/collections/steps/records/:id` | `PATCH /api/collections/goals/records/:id` |
| Delete    | `DELETE /api/collections/steps/records/:id` | `DELETE /api/collections/goals/records/:id` |

**Filtering examples:**
- Steps for a month: `?filter=date >= '2026-03-01' && date <= '2026-03-31'`
- Steps for a year: `?filter=date >= '2026-01-01' && date <= '2026-12-31'`
- Goal for a year: `?filter=year = 2026`

No auth rules needed — all endpoints are public (single user, local network).

### Views (SPA with hash-based routing)

| View           | Route          | Description                                         |
| -------------- | -------------- | --------------------------------------------------- |
| **Dashboard**  | `#/`           | Today's steps, quick entry/edit, current month summary |
| **Month View** | `#/month`      | Calendar grid, monthly stats, progress bar           |
| **Year View**  | `#/year`       | Month-by-month breakdown table + chart               |
| **Stats**      | `#/stats`      | All-time statistics                                  |
| **Settings**   | `#/settings`   | Set yearly goals                                     |

### Dashboard Features
- Large step count display for today
- Quick add/edit today's steps
- Progress ring showing daily goal %
- Mini summary: steps this week, month-to-date

### Month View Features
- Calendar grid with step counts per day
- Color-coded days (goal met = green, missed = red, no data = gray)
- Monthly stats panel:
  - **% completion** — total steps so far / monthly goal
  - **Diff to completion** — steps remaining to hit monthly goal
  - **Ahead/behind** — compared to where you should be by today
  - **Steps/day to complete** — remaining steps / remaining days in month
  - **Completed days** — count of days meeting daily goal
- Bar chart: steps per day for the month

### Year View Features
- Table with one row per month showing:
  - Total steps
  - Average steps/day
  - % of monthly goal
  - Completed days (days meeting daily goal)
- Bar chart: monthly totals
- Year-to-date summary vs. yearly goal

### Stats Features
- **Average steps/day per month** — table + line chart
- **Best week** — Mon-Sun window with highest total, showing dates and total
- **Max in a day** — date and count
- **Biggest streak** — longest consecutive run of days meeting daily goal, showing start/end dates and length

## Computation Details

### Daily Goal
```
daily_goal = yearly_target / days_in_year(year)
```

### Monthly Progress
```
monthly_goal       = daily_goal × days_in_month
current_expected   = daily_goal × current_day_of_month
actual_total       = sum(steps for month)
pct_completion     = actual_total / monthly_goal × 100
diff_to_completion = monthly_goal - actual_total
ahead_behind       = actual_total - current_expected
steps_per_day_remaining = (monthly_goal - actual_total) / remaining_days
completed_days     = count(days where steps >= daily_goal)
```

### Best Week
- Iterate all Monday-to-Sunday windows that have data
- Sum steps for each complete 7-day window
- Return the window with the highest sum

### Biggest Streak
- Sort all step records by date ascending
- Walk through, tracking consecutive days where `count >= daily_goal`
- Track the longest run

## Deployment

### Dockerfile
```dockerfile
FROM alpine:latest

ARG TARGETOS=linux
ARG TARGETARCH=arm64

# Download PocketBase
ADD https://github.com/pocketbase/pocketbase/releases/download/v0.25.x/pocketbase_0.25.x_${TARGETOS}_${TARGETARCH}.zip /tmp/pb.zip
RUN unzip /tmp/pb.zip -d /pb && rm /tmp/pb.zip

# Copy frontend
COPY pb_public /pb/pb_public

# Data volume for SQLite persistence
VOLUME /pb/pb_data

EXPOSE 8080

CMD ["/pb/pocketbase", "serve", "--http=0.0.0.0:8080"]
```

### docker-compose.yml
```yaml
services:
  step-stats:
    build: .
    ports:
      - "8080:8080"
    volumes:
      - pb_data:/pb/pb_data
    restart: unless-stopped

volumes:
  pb_data:
```

### First Run
1. `docker compose up -d`
2. Open `http://<raspberry-pi-ip>:8080/_/` to access PocketBase admin UI
3. Create admin account
4. Collections (`steps`, `goals`) will be auto-created via migration or manually via admin UI
5. App is available at `http://<raspberry-pi-ip>:8080/`

## Development Notes

### Development Workflow
1. Download PocketBase binary for your OS from [pocketbase.io](https://pocketbase.io)
2. Run `./pocketbase serve` from the project root — it serves `pb_public/` automatically
3. Edit frontend files in `pb_public/` — refresh browser to see changes (no build step)
4. Collections can be set up via admin UI (`/_/`) or via migration files

### Key Design Decisions
- **No build step**: Alpine.js + Tailwind CDN + Chart.js CDN means zero tooling. Edit HTML/JS, refresh browser.
- **No auth**: Single-user app on local network. PocketBase API rules set to public access.
- **All computation in frontend**: Stats are calculated client-side. The dataset is small (365 records/year), so this is efficient and keeps the backend simple.
- **Hash-based routing**: Simple SPA navigation without needing a router library. Alpine.js handles view switching.
- **PocketBase migrations**: Schema defined in migration files so the Docker container self-initializes on first run.

### Milestones
1. **M1 — Scaffold**: Dockerfile, PocketBase setup, migration for collections, basic HTML shell
2. **M2 — CRUD**: Step entry form, list view, edit/delete for steps + goal setting

4. **M4 — Month View**: Calendar grid, monthly stats, bar chart
